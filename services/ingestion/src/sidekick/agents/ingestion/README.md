# Ingestion Worker (Scrapy Spiders)

**Role**: Fetch new items from known sources on schedule; write raw artifacts to the store. Fully deterministic — no LLM at crawl time.

## Framework

Scrapy via `CrawlerProcess`, wired in `spiders/_runner.py`. Each source has a committed `SidekickSpider` subclass generated by the examination agent. The spider harness provides:

| Component | File | Purpose |
|-----------|------|---------|
| `ArtifactWriterPipeline` | `spiders/_pipeline.py` | Converts `RawItem`s to `Artifact` rows; binary content (PDF, audio, video) to object storage, text inline. Drops items without `format_id`. |
| `DeduplicationMiddleware` | `spiders/_middleware.py` | Loads already-ingested URLs from artifact store on `spider_opened`; drops duplicates within and across runs. |
| `run_spider()` / `run_spiders()` | `spiders/_runner.py` | Wires `CrawlerProcess` with infrastructure; updates source health (`last_checked`, `last_new_item`, `status`) after every run. |

## Trigger

- **Scheduled**: `sidekick spiders due` — runs all spiders whose cron schedule is currently due
- **On demand**: `sidekick spiders run SOURCE_ID`

## Non-obvious behavior

- Only runs for sources with `examination_status = active` (set by `sidekick spiders sync`).
- Dedup state is loaded from the artifact store at spider open — compares against `source-url` entities in existing `stage=raw` artifacts for the source.
- Health fields are always written back after a run, even on failure (`status` flips to `error` if the spider raises).
- Yields one `RawItem` per asset — PDFs, audio, and video on the same page are separate items.
- Async formats (HLS, mpeg-ts) produce stub artifacts with `status="pending_acquisition"`; an acquisition worker (future) completes them.
- Spider files import only `scrapy` and `sidekick.spiders._base` — no LLM calls inside a spider.
